# 🎯 Аналитика ставок

Streamlit-дашборд для анализа эффективности прогнозов с каналов ставок.

**Главные вопросы, на которые отвечает дашборд:**
1. Какие каналы прогнозов реально зарабатывают?
2. Какие типы ставок сливают?

## Файлы

- `app.py` — главный файл приложения
- `template_stavki.xlsx` — пустой шаблон для заполнения (с dropdown-валидацией)
- `test_data_5days.xlsx` — тестовые данные (59 ставок за 5 дней) для проверки
- `build_template.py` — скрипт, который генерирует шаблон (если надо обновить категории)
- `requirements.txt` — зависимости

## Структура шаблона

**Обязательные колонки** (синие в заголовке):
| Колонка | Что |
|---|---|
| дата | дата матча (YYYY-MM-DD) |
| канал | откуда прогноз — главная ось аналитики |
| спорт | вид спорта |
| лига | лига/чемпионат |
| матч | "Команда1 — Команда2" |
| тип_ставки | "Броски в створ, фора" и т.д. — главная ось типов |
| направление | больше/меньше, плюс/минус, П1/П2 |
| ставка | в рублях |
| кэф | коэффициент ставки |
| результат | WIN / LOSE / RETURN / PENDING |
| профит | считается формулой автоматически |

**Опциональные** (серые в заголовке):
- линия, кэф_прогноза, комиссия, формат, заметки

## Запуск локально

```bash
pip install -r requirements.txt
streamlit run app.py
```

Откроется браузер на http://localhost:8501. Загружай `test_data_5days.xlsx` для демки или свой файл по шаблону.

## Логика категорий

В шаблоне `тип_ставки` — это объединение рынка и типа расчёта в одну колонку. Это сделано осознанно: в реальной аналитике "фора по броскам" — это единое ментальное понятие, а не отдельные оси. Так удобнее группировать и видеть, что работает.

Если нужны более тонкие срезы (отдельно фора, отдельно тотал) — можно добавить в шаблон ещё колонок или расщеплять `тип_ставки` в дашборде.

## Деплой на Railway

1. Запушить репозиторий на GitHub.
2. Railway → New Project → Deploy from GitHub.
3. Settings → Deploy → Start Command:
   ```
   streamlit run app.py --server.port $PORT --server.address 0.0.0.0
   ```

Или проще — Streamlit Cloud (https://streamlit.io/cloud): подключаешь репозиторий, всё автоматом.
